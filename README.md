# Developer Portfolio Analyzer

## Coding Agent

Before making code changes:

1. Read [docs/PROJECT_INDEX.md](docs/PROJECT_INDEX.md) first.
2. Open the relevant feature document from `docs/features/`.
3. Follow [docs/agent/AGENT_RULES.md](docs/agent/AGENT_RULES.md).
4. Update documentation if architecture, APIs, routes, caching, models, services, or feature behavior changes.

For a one-file project map, read [docs/agent/CODING_AGENT_CONTEXT.md](docs/agent/CODING_AGENT_CONTEXT.md).

Before merging documentation or feature-flow changes, run:

```powershell
node docs\verify-docs.js
```
