# Backend

## Interview Prep Question Engine

The Interview Prep module now runs as a self-growing engine with topic buckets, DB-first retrieval, AI/scrape fallback enrichment, and persistence of newly learned Q&A.

### Core Design

- Controller layer: `src/controllers/interviewPrepController.js`
- Service layer: `src/services/interviewPrepService.js`
- Repository layer: `src/repositories/interviewQuestionRepository.js`
- Provider layer:
  - AI: `src/services/providers/interviewAIProvider.js`
  - Scraper: `src/services/providers/interviewScraperProvider.js`
- Topic normalization: `src/services/interviewTopicNormalizer.js`
- Enrichment orchestrator: `src/services/interviewEnrichmentOrchestrator.js`

### Topic Bucket Model

Each question is mapped to:

- `topicKey` (canonical key)
- `topicType` (`stack | technology | language | framework`)
- `topicDimensions` (bucketed dimensions)

Alias normalization examples:

- `Node.js`, `nodejs`, `Node JS` -> `nodejs`
- `C++`, `cplusplus`, `c plus plus` -> `cpp`

### Retrieval and Enrichment Flow

1. Query MongoDB first.
2. If topic pool is sufficient, return DB results only.
3. If insufficient:
   - AI generation fills missing amount.
   - Scraping fills remaining gap.
4. Validate + dedupe + save.
5. Return merged page with metadata.

Response metadata includes:

- `totalAvailable`
- `enrichedCount`
- `sourceMix`
- `fromCache`
- `partial`
- `aiGeneratedCount`
- `scrapedGeneratedCount`

### Custom Ask Flow

`POST /api/interview-prep/ask-question`

Behavior:

1. Answer is generated by AI.
2. Topic is normalized to canonical bucket.
3. Q&A is deduplicated and saved with:
   - `sourceType = user_asked`
   - tags include `ai_generated`, `user_asked`

### Data Quality Controls

Before saving records:

- Question/answer normalization
- Minimum quality checks
- Unsafe content filtering
- Exact + fuzzy deduplication
- Confidence scoring

### Background Jobs

1. Scrape ingestion scheduler:
   - `src/services/interviewQuestionIngestionService.js`
2. Minimum-pool maintenance scheduler:
   - `src/services/interviewQuestionMaintenanceService.js`

Maintenance target defaults to 20 per important topic.

### Scripts

Run from `backend`:

- `npm run migrate:interview-bank`
  - Backfills topic fields, normalized fields, removes duplicates, ensures new indexes.
- `npm run seed:interview-bank`
  - Seeds at least 20 prebuilt questions per important topic.

### API Summary

- `GET /api/interview-prep/questions`
- `GET /api/interview-prep/search`
- `POST /api/interview-prep/generate`
- `POST /api/interview-prep/ask-question`
- `POST /api/interview-prep`

Backward compatibility is preserved for existing `skill`, pagination, and filter behavior.
